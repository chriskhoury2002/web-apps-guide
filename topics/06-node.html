צד שרת ו-Node.js

עד עכשיו עבדנו בצד הלקוח. עכשיו נעבור לצד השרת. נכיר את Node.js, סביבה שמאפשרת להריץ JavaScript מחוץ לדפדפן, נבין מה זה מודול, ונבנה שרת HTTP ראשון.

שפות לצד השרת

בצד השרת (back-end) אפשר לתכנת בהרבה שפות: Node.js (JavaScript), PHP, ASP.NET‏ (C#), JSP‏ (Java), ו-Python (עם Django). בקורס נתמקד ב-Node.js, כי ככה נשתמש באותה שפה, JavaScript, גם בצד הלקוח וגם בצד השרת. זה נקרא "full-stack JS".

מה זה Node.js

Node.js נוצר ב-2009 על ידי Ryan Dahl. זו סביבת הרצה שמאפשרת להריץ קוד JavaScript מחוץ לדפדפן. עיקרי הדברים:

  • השימוש העיקרי הוא לבניית שירותי צד שרת (back-end).
  • הוא מהיר וניתן להרחבה (scalable), ומריץ קוד ב-thread יחיד.
  • הוא עובד עם event loop ולא חוסם (non-blocking), ולכן מומלץ להימנע מקוד סינכרוני שחוסם את ה-thread היחיד.
  • חברות גדולות משתמשות בו: Uber, PayPal, Walmart, Netflix ועוד.
  • הוא לא מתאים למשימות שדורשות הרבה כוח חישוב (CPU intensive), כי thread יחיד ייתקע. הוא מצוין למשימות עם הרבה בקשות אבל מעט חישוב.
  • אפשר לארח את ה-back-end בענן (cloud based).

בדפדפן יש גישה ל-DOM (document), ב-Node אין DOM אבל יש גישה למערכת הקבצים, לרשת ועוד.

מודולים: module.exports ו-require

ב-Node כל קובץ .js הוא מודול. כדי לחשוף משתנים או פונקציות מחוץ למודול, מציבים אותם ב-module.exports. כדי להשתמש במודול אחר, טוענים אותו עם require.

מודולים מובנים

ל-Node יש מודולים מובנים שימושיים, שטוענים עם require:

  • os, גישה לפונקציות של מערכת ההפעלה.
  • fs, עבודה עם קבצים (קריאה, כתיבה).
  • events, לולאת אירועים.
  • http, שירות בקשות HTTP (בניית שרת).
