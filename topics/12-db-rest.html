מסד נתונים MySQL ו-REST

אפליקציה אמיתית צריכה לשמור נתונים. נלמד לחבר Node ל-MySQL, להריץ שאילתות, ולקרוא פרמטרים מבקשות. אחר כך נבין את REST, את המיפוי בין מתודות HTTP לפעולות CRUD, ואת רעיון המיקרו-שירותים.

REST, מתודות HTTP ו-CRUD

REST (REpresentational State Transfer) הוא סגנון ארכיטקטוני לתקשורת בין לקוח לשרת. שירותים שמצייתים לו נקראים RESTful. המפתח: מתודות ה-HTTP ממופות לפעולות CRUD על נתונים. לחץ על מתודה וראה מה היא עושה:

🔗 service מול site

service מיועד לתוכניות (מחזיר נתונים, למשל JSON), ו-site מיועד למשתמש אנושי (מחזיר HTML). API הוא service.

מיקרו-שירותים

בארכיטקטורת מיקרו-שירותים האפליקציה היא אוסף של שירותים קטנים ומקושרים באופן רופף. כל שירות:

  • קטן, אפשר לשכתב אותו בשבוע-שבועיים.
  • ממוקד במשימה אחת.
  • עצמאי, אפשר לשנות אותו בלי לתאם עם אחרים.
  • ניתן לפריסה בנפרד.
  • מקושר רופף, הממשק שלו הוא API ציבורי.
