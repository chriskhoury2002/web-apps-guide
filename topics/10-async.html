אסינכרוניות, Fetch ו-JSON

JavaScript רץ ב-thread יחיד, אז איך הוא מטפל בהמון משתמשים בו-זמנית בלי להיתקע? התשובה היא תכנות אסינכרוני. בנושא הזה נבין את ה-event loop, נראה את ההבדל בין קוד חוסם ללא חוסם, ונלמד callbacks, promises, async/await, fetch ופורמט JSON. זה אחד הנושאים החשובים ביותר בקורס.

Thread יחיד ו-Event Loop

JavaScript הוא single-threaded: רק דבר אחד יכול לקרות בכל רגע. אם פעולה אחת נתקעת, הכל נתקע. אז איך Node.js משרת אלפי משתמשים? במקום ליצור thread לכל משתמש (כמו PHP או Java, וזה יקר), Node עובד עם Event Loop.

הרעיון: פעולות איטיות (קריאת קובץ, פנייה למסד נתונים, בקשת רשת) לא מבוצעות ב-thread הראשי. הן נמסרות ל-Web API שמטפל בהן ברקע, ורק כשהן מסתיימות ה-callback שלהן נכנס לתור וממתין לתורו. ה-Event Loop הוא הלולאה שבודקת מתי המחסנית ריקה ומכניסה אליה את ה-callback הבא.

💡 למה זה חכם?

רוב הזמן בשרת עובר בהמתנה (לדיסק, לרשת, ל-DB), לא בחישוב. במקום ש-thread ישב ויחכה, Node משחרר אותו לטפל בבקשות אחרות, וחוזר ל-callback רק כשהתוצאה מוכנה. ככה thread יחיד משרת המון בקשות.

הדוגמה הקלאסית: מה סדר ההדפסות בקוד הבא? התשובה מפתיעה. הפעל את האנימציה וראה בדיוק איך ה-Event Loop עובד עם המחסנית והתורים:

⚡ חוק הזהב

אחרי שהמחסנית מתרוקנת, ה-Event Loop מרוקן קודם את כל ה-Microtask Queue (של Promises) ורק אחר כך לוקח פריט אחד מה-Callback Queue (של setTimeout ואירועים). לכן `Promise.then` תמיד ירוץ לפני `setTimeout`, גם אם ה-setTimeout היה עם 0 מילי-שניות.

חוסם מול לא חוסם (Blocking vs Non-Blocking)

נניח שרוצים לקרוא קובץ ואז להציג אותו. יש שתי דרכים. בגרסה החוסמת (סינכרונית) ה-thread ממתין עד שהקובץ נקרא, ורק אז ממשיך. בגרסה הלא-חוסמת (אסינכרונית) שולחים בקשה לקרוא, ממשיכים לעבוד, וכשהקובץ מוכן רץ ה-callback.

הפעל את האנימציה כדי לראות למה הגרסה הלא-חוסמת מסיימת מהר יותר:

Callbacks

Callback היא פונקציה שמעבירים לפונקציה אחרת, כדי שתופעל מאוחר יותר כשמשהו קורה. הרבה פעולות אסינכרוניות מבוססות על callbacks:

Promises ו-then

פונקציה async מחזירה אובייקט Promise (הבטחה). ל-Promise יש מתודה .then שמקבלת פונקציה שתרוץ כשההבטחה מתקיימת, עם הערך שהוחזר. אפשר לשרשר כמה .then אחד אחרי השני, וזה הרבה יותר קריא ממדרגות של callbacks:

async / await

await הוא תחביר נוח יותר במקום .then. הכלל: await אפשר להשתמש רק בתוך פונקציית async. כשמגיעים ל-await, הפונקציה הנוכחית "נעצרת" ומחזירה שליטה, וממשיכה משם ברגע שהפעולה האסינכרונית שאחרי ה-await מסתיימת.

הנקודה הכי מבלבלת ב-await היא סדר ההרצה. בוא נראה בדיוק מה קורה, צעד אחר צעד. שים לב איך B מודפס לפני C:

Fetch: להביא נתונים מהשרת

fetch היא הדרך המודרנית לשלוח ולקבל נתונים מהשרת בעזרת JavaScript, בלי לטעון מחדש את הדף. היא מחליפה שיטות ישנות כמו AJAX ו-XMLHttpRequest. דוגמאות מוכרות: הצעות ההשלמה של גוגל בזמן הקלדה, וטעינת חלקים חדשים במפה של Google Maps כשגוררים אותה.

אובייקט ה-response שחוזר מ-fetch כולל כמה תכונות שימושיות:

תכונהמה היא
response.statusקוד הסטטוס (למשל 200, 404)
response.okהאם הבקשה הצליחה (סטטוס בטווח 2xx)
response.statusTextטקסט הסטטוס (למשל "OK")
response.json()מפענח את גוף התשובה כ-JSON

לשלוח נתונים לשרת (POST)

כדי לשלוח נתונים מוסיפים אובייקט הגדרות עם method, headers ו-body. את הגוף הופכים למחרוזת JSON:

JSON

JSON (JavaScript Object Notation) הוא פורמט טקסט קליל להעברת נתונים, קל לקריאה גם לאדם וגם למחשב. הוא עצמאי משפה, אבל התחביר שלו לקוח מ-JavaScript. היום הוא הפורמט הכי נפוץ להעברת מידע ברשת, והוא החליף את XML.

JSON הוא תת-קבוצה של תחביר האובייקטים של JS, עם שני הבדלים חשובים: מחרוזות חייבות מרכאות כפולות, וגם שמות השדות חייבים מרכאות.

שתי הפונקציות החשובות להמרה בין מחרוזת JSON לאובייקט JS:

  • JSON.parse(), ממחרוזת JSON לאובייקט.
  • JSON.stringify(), מאובייקט למחרוזת JSON.
