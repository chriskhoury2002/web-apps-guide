יסודות CSS

אם HTML הוא השלד, CSS הוא הלבוש. CSS קובע איך הדף נראה: צבעים, גדלים, מרווחים, מיקום. בנושא הזה נלמד לכתוב כללי עיצוב, נבין את מודל הקופסה, ונראה איך סלקטורים "תופסים" בדיוק את האלמנטים שרוצים לעצב.

מה זה CSS ולמה משתמשים בו

CSS הוא ראשי תיבות של Cascading Style Sheets, "גיליונות סגנון מדורגים". תפקידו להגדיר איך אלמנטים של HTML נראים על המסך. היתרון הגדול: אפשר לשלוט בעיצוב של דפים רבים בבת אחת, ממקום אחד.

💡 למה מפרידים בין תוכן לעיצוב?

כשהתוכן (HTML) נפרד מהעיצוב (CSS), אפשר לשנות את כל המראה של האתר בלי לגעת בתוכן, ולהשתמש באותו עיצוב בהרבה דפים. זה חוסך עבודה ומונע בלגן. זה הרעיון של "הפרדת אחריות".

שלושה מקומות להגדיר CSS

אפשר לכתוב CSS בשלוש דרכים, מהגרועה לטובה:

  • בשורה (inline), בתוך מאפיין style של אלמנט בודד. לא מומלץ: אין שימוש חוזר ואין הפרדה בין עיצוב לתוכן.
  • פנימי (internal), בתוך תגית <style> ב-<head>. טוב יותר, אבל משרת רק מסמך אחד.
  • חיצוני (external), קובץ .css נפרד שמקשרים אליו עם <link>. הכי טוב: רמה גבוהה של שימוש חוזר בין דפים.
🔀 סדר הקדימות (Cascade)

אם יש כמה הגדרות לאותו אלמנט, הן "נערמות" לפי סדר עדיפות, מהנמוך לגבוה: ברירת מחדל של הדפדפן, ואז חיצוני, ואז פנימי, ואז inline. כלומר inline "מנצח" את כולם. זו המשמעות של המילה Cascading.

תחביר של כלל CSS

כלל CSS בנוי משלושה חלקים: selector (על מי הכלל חל), property (איזו תכונה משנים) ו-value (הערך).

אפשר להגדיר כמה תכונות, ומפרידים ביניהן בנקודה-פסיק. כאן ה-selector הוא h1, ויש שתי הצהרות: צבע וגודל גופן.

סלקטורים: element, class, id

איך אומרים ל-CSS על אילו אלמנטים לחול? עם סלקטור. שלושת הבסיסיים:

  • Element selector, שם של תגית. למשל p חל על כל הפסקאות.
  • Class selector, מתחיל בנקודה. למשל .center חל על כל אלמנט עם class="center". אפשר לשים את אותה class על הרבה אלמנטים.
  • Id selector, מתחיל בסולמית. למשל #para1 חל על האלמנט עם id="para1". id חייב להיות ייחודי בדף, ולכן חל על אלמנט אחד בלבד.

הכי טוב ללמוד סלקטורים דרך ניסוי. לחץ על סלקטור וראה אילו אלמנטים הוא תופס:

✅ class מול id

משתמשים ב-class כשרוצים לעצב קבוצה של אלמנטים, וב-id כשמדובר באלמנט יחיד וספציפי. בפועל class הרבה יותר נפוץ בעיצוב.

מודל הקופסה (Box Model)

כל אלמנט ב-HTML הוא בעצם קופסה מלבנית, ולקופסה יש ארבע שכבות, מבפנים החוצה:

  • content, התוכן עצמו (טקסט, תמונה).
  • padding, ריפוד, המרווח בין התוכן למסגרת.
  • border, המסגרת סביב הריפוד.
  • margin, המרווח החיצוני בין הקופסה לאלמנטים אחרים.

הבנת המודל הזה חיונית לכל עיצוב. שחק עם המחוונים וראה איך כל שכבה משפיעה על הגודל הכולל:

📐 חישוב הגודל

הרוחב הכולל של קופסה (כברירת מחדל) הוא: רוחב התוכן ועוד padding משני הצדדים ועוד border משני הצדדים. ה-margin לא נספר בתוך הקופסה, אלא מוסיף מרווח מסביבה.

תכונות טקסט

כמה מהתכונות הנפוצות לעיצוב טקסט:

תכונהמה עושהדוגמת ערכים
colorצבע הטקסטred, #333
text-alignיישור אופקיright, center, justify
text-decorationקו תחתי/עליון/חוצהunderline, none
text-transformגודל אותיותuppercase, capitalize
letter-spacingמרווח בין אותיות2px
directionכיוון כתיבהrtl, ltr

תכונות גופן

  • font-family, רשימת גופנים לפי עדיפות. הדפדפן ישתמש בראשון שקיים.
  • font-size, גודל (למשל 16px, large, 120%).
  • font-style, נטוי (italic) או רגיל.
  • font-weight, עובי (bold, 400, 700).

מסגרת, margin ו-padding

אלה התכונות ששולטות במסגרת ובמרווחים סביב האלמנט:

  • border, מסגרת. למשל border: 2px solid black; (עובי, סגנון, צבע). סגנונות נפוצים: solid, dashed, dotted.
  • padding, ריפוד פנימי בין התוכן למסגרת.
  • margin, מרווח חיצוני. טריק שימושי: margin: auto; ממרכז אלמנט אופקית.

תכונות רשימה

אפשר לשלוט בסימון של רשימות עם list-style-type. למשל disc (נקודה), circle, square, decimal (מספרים) או none (בלי סימון בכלל, שימושי מאוד לתפריטים).
