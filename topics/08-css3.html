CSS3 מתקדם

CSS3 הביא כלים חזקים לפריסה ולעיצוב: סלקטורים מתקדמים, מיקום, ובעיקר Flexbox ו-Grid לפריסות גמישות ורספונסיביות. נבין גם איך מוכרעת התנגשות בין כללים (specificity), נושא שחוזר הרבה במבחנים.

מה זה CSS3

CSS3 נחשב חלק מ-HTML5. שלא כמו גרסאות קודמות, הוא מחולק למודולים נפרדים (למשל Selectors, Flexbox, Grid), שכל אחד מתפתח בקצב שלו וקיבל רמות (levels) 1 עד 4.

סלקטורים מתקדמים

מלבד סלקטורים בסיסיים (element, class, id), CSS3 מוסיף:

  • Combinators: div p (צאצא בכל עומק), div > p (ילד ישיר), div + p (האח הבא מיד), div ~ p (כל האחים אחריו).
  • Attribute selectors: [target], a[href^="https"] (מתחיל ב), a[href$=".pdf"] (מסתיים ב), a[href*="w3c"] (מכיל).

Pseudo-classes ו-Pseudo-elements

Pseudo-classes (עם נקודתיים אחת) בוחרים אלמנט לפי מצב: :hover, :focus, :checked, :first-child, :nth-child(2), :not(...), :valid, :invalid.

Pseudo-elements (עם נקודתיים כפולות) מעצבים חלק מאלמנט: ::before, ::after, ::first-letter, ::first-line, ::selection.

חוזק סלקטור (Specificity), מי מנצח?

זה נושא קריטי. כשכמה כללים חלים על אותו אלמנט וסותרים זה את זה, מכריעים לפי specificity (חוזק הסלקטור). סדר החוזק מהחלש לחזק:

סלקטורדוגמהחוזק
element (שם תגית)pהכי חלש
class (נקודה).boxבינוני
id (סולמית)#mainחזק
inline stylestyle="..."הכי חזק

כלל נוסף: בין שני כללים עם אותו חוזק, זה שמופיע מאוחר יותר בקוד מנצח.

נסה בעצמך. הטקסט אדום כי סלקטור ה-id (#a) חזק מכולם. מחק את הכלל של #a וראה שהוא הופך לירוק (class), ואז מחק גם את .b ותקבל שחור (element):

⚠️ טעות נפוצה במבחן

הרבה נופלים על "מי מנצח". זכור: id חזק מ-class, ו-class חזק מ-element. inline חזק מכולם. ובשוויון, המאוחר בקוד מנצח.

מיקום (Position)

  • static, ברירת המחדל, זרימה רגילה.
  • relative, יחסית למיקום הרגיל.
  • fixed, יחסית לחלון התצוגה, לא זז בגלילה.
  • absolute, יחסית לאב הממוקם הקרוב, זז עם הגלילה.
  • sticky, כמו relative, אבל נדבק למסך בגלילה.

עם z-index שולטים על מי מעל מי: ערך גבוה יותר נמצא מלפנים.

Flexbox

Flexbox הוא מודל פריסה חד-ממדי (שורה או עמודה). מגדירים display: flex על המיכל, ואז שולטים במיקום עם justify-content (ציר ראשי), align-items (ציר משני), flex-direction ו-flex-wrap. שחק עם הכפתורים וראה איך הפריטים זזים:

עיצוב רספונסיבי

עיצוב רספונסיבי (responsive design) דואג שהדף ייראה טוב על כל גודל מסך: טלפון, טאבלט, מחשב. הכלים המרכזיים:

  • Media Queries, כללי CSS שחלים לפי רוחב המסך (breakpoints).
  • יחידות גמישות: rem, fr, אחוזים, במקום פיקסלים קבועים.
  • Mobile-first, מתחילים מהמסך הקטן ומרחיבים.
  • תג ה-viewport, חיוני כדי שהעיצוב יעבוד במובייל.
📱 בלי viewport אין רספונסיביות

תג ה-viewport אומר לדפדפן במובייל להתאים את רוחב הדף לרוחב המכשיר. בלעדיו, גם עם media queries, הדף ייראה מוקטן ולא מותאם.
